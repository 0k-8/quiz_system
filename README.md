# Quiz System — Infrastructure Layer 🚀

Система для проведення командних квізів, розроблена на Java з використанням JDBC та архітектурного патерну DAO (Data Access Object). Проєкт фокусується на надійному управлінні даними та ефективному використанні ресурсів бази даних.

## 📌 Огляд проєкту

Даний етап розробки присвячений реалізації **Infrastructure Layer**. Система дозволяє керувати користувачами, командами, тестами (квізами), питаннями та результатами проходження.

### Ключові особливості:
* **Custom Connection Pool**: Власна реалізація пулу з'єднань для оптимізації роботи з БД.
* **Automated Migrations**: Використання Flyway для управління схемою бази даних.
* **Data Integrity**: Чітка ієрархія зв'язків (Foreign Keys) між командами, гравцями та результатами.
* **Flexible Configuration**: Налаштування через `application.properties`.

---

## 🛠 Технологічний стек

* **Language:** Java 22
* **Database:** H2 Database (Embedded/Server mode)
* **Migration Tool:** Flyway
* **Libraries:** JDBC, Jackson (JSON), Lombok (optional)
* **Build Tool:** Maven

---

## 🏗 Архітектура бази даних

Проєкт використовує наступні сутності:
1.  **Teams** — Команди учасників.
2.  **Users** — Користувачі з ролями (ADMIN, USER).
3.  **Quizzes** — Тести з обмеженням за часом.
4.  **Questions** — Питання з варіантами відповідей (A, B, C, D).
5.  **Results** — Командні результати проходження квізів.

---

## 🚀 Тестування (Infrastructure Layer)

Для перевірки працездатності шару інфраструктури було реалізовано наскрізний інтеграційний тест у класі `Main`.

### Сценарій тестування:
1.  **Ініціалізація**: Автоматичне зчитування налаштувань з `application.properties` та створення `ConnectionPool`.
2.  **CRUD операції**: Послідовне створення та збереження взаємопов'язаних об'єктів.
3.  **Перевірка зв'язків**: Валідація того, що питання прив'язані до квізів, а результати — до команд.
4.  **Вибірка даних**: Тестування методів `findAll()`, `findById()` та логіки топ-результатів.
